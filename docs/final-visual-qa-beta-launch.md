# Final Visual QA & Beta Launch Checklist v1

## 1. Color system check
- [ ] Background feels clean, not dull.
- [ ] Cards are separated without heavy borders.
- [ ] Primary CTA remains deep navy.
- [ ] Text contrast is strong.
- [ ] No flashy colors.

## 2. Public landing check
- [ ] Visitor understands within 5 seconds: photo/problem input → OCR draft → skeleton/explanation → next action.
- [ ] Today input CTA remains primary.
- [ ] Answer Review Studio CTA is visible but secondary.
- [ ] Skeleton demo is clearly a learning-aid draft.

## 3. Capture check
- [ ] 사진 찍기 is the dominant mobile action.
- [ ] OCR draft copy is visible.
- [ ] Textarea is comfortable.
- [ ] Optional metadata is secondary.

## 4. Answer Review Studio check
- [ ] 답안 스냅으로 시작 opens camera where supported.
- [ ] 사례 스캔 opens camera/file picker.
- [ ] 텍스트 붙여넣기 focuses textarea.
- [ ] 결과 화면 shows:
  - [ ] 가장 큰 간극
  - [ ] 누락 논점
  - [ ] 약한 구조
  - [ ] 다시 쓸 문장
  - [ ] 다음 행동

## 5. Mobile check
- [ ] No horizontal overflow.
- [ ] CTA buttons have enough tap area.
- [ ] Cards stack cleanly.
- [ ] Hero demo does not become too long or cramped.

## 6. Safety check
- [ ] No official grading claims.
- [ ] No pass/fail claims.
- [ ] No guaranteed score/pass claims.
- [ ] No new OCR provider scope.
- [ ] No instructor/admin route exposure.

## 7. Launch decision
- Status: **Pending manual visual QA**

  - invited users only
  - 감정평가사 1차
  - 감정평가사 2차
  - no payment
  - no broad public launch
