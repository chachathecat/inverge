# Inverge Legal Retrieval

## Purpose

Legal Retrieval v1 adds a safe source-anchor layer for future 민법 and 관계법규 explanations. It returns official law article candidates from `legal_article_chunks`; it does not generate learner-facing explanations.

## Source Boundary

Official law chunks are source anchors, not official model answers. A retrieved article can support later explanation drafting, but it is not grading, not a pass/fail judgment, and not an official answer key.

No source, no legal claim. If retrieval cannot find a source anchor, legal explanation generation must stop or clearly mark the point as unsupported.

Legal retrieval is a precondition for 민법/관계법규 explanations. Future explanation code should first retrieve article anchors, then apply the grounding policy before drafting any legal claim.

## Data Separation

Learner raw input remains separate from the legal corpus:

- Do not store raw user OCR text in `legal_sources`, `legal_versions`, or `legal_article_chunks`.
- Do not store raw learner answers or problem text in the legal corpus.
- Do not store academy materials, official grading claims, official model answers, score prediction, or pass/fail judgments in the legal corpus.

## Current Retrieval

`search_legal_chunks_keyword` is an authenticated, read-only RPC over current law versions. It returns article-level candidates with `article_key`, display article fields, body text, metadata, and a keyword rank score.

The application helper returns an explicit empty state when no anchor is found:

```ts
{
  grounded: false,
  reason: "no_source_anchor_found",
  candidates: []
}
```

## Future Work

Vector retrieval can be added later after embeddings are generated by a separate approved pipeline. The same grounding rule still applies: no source, no legal claim.
