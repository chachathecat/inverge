# Inverge Ethical Retention Loop

## 2026-07-01 Product Constitution Transition

Retention for Dabangil must be ethical retention only: anxiety relief, morning brief, due review, recovery nudge, and capture reminder. The product must not use shame copy, ranking pressure, fake urgency, intentionally degraded free output, casino-style gamification, addictive streak mechanics, or hidden answer access designed only to force payment.

Every nudge should reduce the learner's next decision to one useful action, preserve easy override, and keep Today Plan max 3.

## Ethical retention principles

Inverge should be useful enough to return to, not addictive enough to trap learners. Retention should come from anxiety relief, visible progress, and reduced decision load.

Allowed principles:

- set a good default next action;
- provide easy override options;
- normalize mistakes as learning input;
- use smart friction only to clarify consequence;
- end feedback with retry, rewrite, or scheduled review;
- keep Today Plan to max 3 primary tasks.

Prohibited patterns:

- shame or fear language;
- ranking pressure;
- fake urgency;
- manipulative scarcity;
- casino-style gamification;
- addictive streak mechanics;
- notification pressure that punishes missed days.

## Anxiety relief loop

The ethical loop is:

1. Learner captures uncertainty or a missed task.
2. Inverge names one biggest gap.
3. Inverge chooses one default next action.
4. Learner completes retrieval or production.
5. Feedback identifies the next retry/rewrite/scheduled review.
6. The queue becomes smaller or clearer.

The emotional outcome should be: “무엇을 하면 되는지 알겠다.”

## Morning brief

Morning brief should be a calm start-of-day plan:

- mention due review without blame;
- show up to three Today Plan tasks;
- include a 30-minute fallback;
- avoid countdown pressure unless the learner explicitly set an exam date;
- let the learner skip or reschedule.

## Due review

Due review should be opt-out by default because spaced retrieval is core learning behavior. Copy should describe due review as maintenance, not punishment.

Example tone: “오늘 고정하면 좋은 복습이 있습니다.”

## Recovery nudge

A missed day should produce recovery, not shame.

Recommended pattern:

- acknowledge the miss neutrally;
- reduce scope;
- offer one low-friction restart;
- preserve the original plan when possible.

Example tone: “어제 못 한 항목은 오늘 1개만 복구해도 충분합니다.”

## Capture reminder

Capture reminders should appear only when they help convert unresolved uncertainty into a next action.

Use when:

- the learner recently marked unknown items;
- O/X wrong items lack concept nodes;
- a 2차 rewrite has missing issue metadata;
- a calculation mistake lacks CASIO sequence tags.

Do not use reminders as generic engagement spam.

## No shame copy

Avoid:

- “또 실패했습니다.”
- “순위가 떨어졌습니다.”
- “지금 안 하면 늦습니다.”
- “연속 기록이 깨졌습니다.”

Prefer:

- “오늘은 범위를 줄여 복구합니다.”
- “가장 큰 틈 하나만 고정합니다.”
- “30분 모드로 바꿀 수 있습니다.”

## No casino-style gamification

Do not use variable rewards, loot-box-like reveals, excessive streak pressure, or random reward mechanics. Progress indicators should show learning coverage, due review load, confidence movement, and retry/rewrite completion.
