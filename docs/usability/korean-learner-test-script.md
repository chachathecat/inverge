# Korean Learner Usability Test Script v1 (감정평가사)

## 목적
- 감정평가사 1차/2차 학습자의 실제 사용 흐름에서 마찰 지점을 관찰한다.
- 기능 성공 여부가 아니라 **행동 기반**으로 다음 개선 우선순위를 정한다.
- 본 문서는 내부 운영용이며, 학습자 정답/문항 원문 수집 없이 진행한다.

## 테스트 설정
- 대상: 감정평가사 학습자 5명 (1차 3명, 2차 2명 권장)
- 진행 시간: 1인당 35~45분
- 기록 방식: 관찰자 1명 + 노트 담당 1명
- 원칙: 개입 최소화, 과제 지시 외 설명 금지

## 공통 관찰 루브릭 (각 과제별 1~5점)
- **clarity**: 사용자가 다음 행동을 스스로 이해했는가?
- **friction**: 어디에서 멈춤/되돌아감/재시도를 했는가?
- **cognitive load**: 화면이 무겁거나 선택지가 과했는가?
- **trust**: AI 결과가 안전하고 검수 가능한 느낌이었는가?
- **study value**: 실제 공부가 되는 경험으로 느꼈는가?
- **aesthetics**: 프리미엄/미니멀 톤으로 인지했는가?

## 진행 스크립트 (5개 과제)

### 과제 1) 첫 온보딩 → 첫 캡처 완료
- 시작 조건: 신규 세션(로그인 완료 상태)
- 과제 문구: “처음 사용자라고 가정하고 첫 학습 캡처를 저장해 주세요.”
- 성공 기준:
  - 온보딩의 다음 행동을 스스로 선택한다.
  - 첫 캡처 저장까지 완료한다.
- 관찰 포인트:
  - 안내 문구 이해 시간
  - 잘못된 진입/뒤로가기 횟수

### 과제 2) 1차 모드 오답 캡처 → 5분 재시도 예약
- 시작 조건: 1차 학습 화면 진입
- 과제 문구: “오답 하나를 캡처하고, 5분 뒤 다시 보도록 재시도를 잡아 주세요.”
- 성공 기준:
  - 오답 캡처 저장
  - 5분 재시도(또는 동등한 즉시 follow-up) 설정
- 관찰 포인트:
  - 저장 후 다음 액션 인지 여부
  - 재시도 설정에서의 망설임

### 과제 3) 2차 모드 서술 흐름 (이슈 회상 → 재작성)
- 시작 조건: 2차 쓰기/리라이트 가능한 상태
- 과제 문구: “쟁점을 먼저 회상한 뒤, 답안을 재작성해 주세요.”
- 성공 기준:
  - 이슈 회상 단계 수행
  - 재작성 제출까지 이어짐
- 관찰 포인트:
  - 회상 우선 흐름이 자연스러운지
  - 피드백 이후 바로 재작성으로 이어지는지

### 과제 4) OCR 실패 → 수동 대체 입력
- 시작 조건: OCR 실패를 재현한 상태(의도적 저품질 이미지 등)
- 과제 문구: “OCR이 실패했다고 가정하고, 수동 방식으로 계속 진행해 주세요.”
- 성공 기준:
  - 막히지 않고 수동 입력 경로를 찾아 진행
  - 학습 루프(저장/재시도)로 복귀
- 관찰 포인트:
  - 실패 메시지의 안정감
  - 대체 경로의 가시성

### 과제 5) 오늘 할 일 완료 → 홈 복귀
- 시작 조건: 오늘 할 일 1개 이상 존재
- 과제 문구: “오늘 할 일을 하나 끝내고 홈으로 돌아가 주세요.”
- 성공 기준:
  - today task 완료 처리
  - 홈에서 다음 액션을 인지
- 관찰 포인트:
  - 완료 후 화면 피로도
  - 다음 액션의 명확성

## 기록 템플릿 (권장)
각 과제마다 아래 항목 기록:
- task
- clarity (1~5)
- friction (1~5)
- cognitiveLoad (1~5)
- trust (1~5)
- studyValue (1~5)
- aesthetics (1~5)
- hesitationPoint (자유기술)
- participantQuote (자유기술)
- suggestedFix (자유기술)

## 안전/프라이버시 가이드
- 정답 원문, 문제 원문, raw OCR 텍스트는 수집하지 않는다.
- 메모는 라우트/행동/마찰 중심으로만 남긴다.
- 관찰 메모는 내부 품질 개선 용도로만 사용한다.
