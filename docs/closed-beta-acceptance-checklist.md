# Inverge Closed Beta Acceptance Checklist

## 목적
실사용자 초대 전, 운영팀/QA가 반드시 수동으로 확인해야 할 핵심 흐름과 데이터 안전성 점검 항목을 표준화한다.

## 범위 및 원칙
- 문서/체크리스트 작성 전용 (동작 변경 없음)
- 다음 항목은 이번 점검에서 변경 금지:
  - 스키마
  - 인증(Auth)
  - Gemini 연동
  - Supabase 연동
  - learning-signal persistence
- 학습자 대상 제품 범위는 **감정평가사 1차/2차**로 한정한다.

## 사전 준비
- 테스트 계정 1개 이상 준비 (로그인 가능 상태)
- 브라우저 개발자도구(Network, Console) 사용 가능
- DB/관리자 확인 권한(읽기 전용 권장)
- 기준 환경(스테이징 또는 베타 후보 배포본) 고정

---

## 1) 1차(First-stage) 플로우 점검

### 1-1. 로그인
- [ ] 로그인 페이지 진입 가능
- [ ] 정상 계정으로 로그인 성공
- [ ] 로그인 후 학습자 앱 컨텍스트로 진입 (공개 마케팅 UI가 섞여 보이지 않음)

### 1-2. 1차 모드 진입
- [ ] ` /app?mode=first `로 직접 진입 가능
- [ ] 화면에서 1차 맥락이 명확히 보임 (2차와 혼동 없음)

### 1-3. 오답 입력 1건 수행
- [ ] 의도적으로 틀린 답안을 1건 입력
- [ ] 제출/확인 이후 결과 화면이 깨지지 않음
- [ ] 다음 액션(재시도/복기 등) 유도가 자연스럽게 노출됨

### 1-4. Today Plan 업데이트 확인
- [ ] 입력 직후 Today Plan(오늘 할 일) 항목 수/상태가 갱신됨
- [ ] 갱신 내용이 방금 수행한 1차 활동과 의미상 일치함

### 1-5. Records 반영 확인
- [ ] Records 페이지에서 방금 입력한 1차 기록이 조회됨
- [ ] 시간순/맥락이 이해 가능하게 표시됨

---

## 2) 2차(Second-stage) 플로우 점검

### 2-1. 2차 모드 진입
- [ ] ` /app?mode=second `로 직접 진입 가능
- [ ] 화면에서 2차 맥락이 명확히 보임 (1차와 혼동 없음)

### 2-2. 작성/비교/재작성(write/compare/rewrite) 경로 점검
- [ ] 작성(Write): 답안 작성 입력이 정상 동작
- [ ] 비교(Compare): 비교 결과가 사용자 관점에서 읽기 가능
- [ ] 재작성(Rewrite): 재작성 액션으로 자연스럽게 이어짐
- [ ] 각 단계에서 주요 CTA가 다음 목적지와 일치함

### 2-3. Today Plan 업데이트 확인
- [ ] 2차 플로우 완료 후 Today Plan이 갱신됨
- [ ] 갱신 내용이 2차 활동을 반영하며 1차 항목과 혼재되지 않음

### 2-4. Records 이해 가능성 확인
- [ ] Records에서 2차 관련 기록이 확인됨
- [ ] 사용자 입장에서 “무엇을 했고 다음에 무엇을 해야 하는지” 이해 가능함

---

## 3) Answer-review 플로우 점검

### 3-1. 유의미 입력 시 learning signal 저장
- [ ] 충분한 길이/의미를 가진 입력으로 리뷰 수행
- [ ] 결과 완료 후 learning signal 저장 이벤트가 확인됨

### 3-2. 불충분 입력 차단
- [ ] 지나치게 짧거나 무의미한 입력으로 시도
- [ ] 시스템이 진행을 차단하고 수정 유도 메시지를 제공
- [ ] 차단 케이스에서는 learning signal이 저장되지 않음

### 3-3. fallback-heavy 결과 저장 방지
- [ ] fallback-heavy로 분류될 수 있는 상황을 재현(예: 모델 응답 품질 저하/대체 응답 경로)
- [ ] 결과는 표시되더라도 learning signal 저장은 발생하지 않음
- [ ] 사용자 메시지는 내부 구현 노출 없이 이해 가능함

---

## 4) 데이터 점검

### 4-1. learning_signal_events 민감 텍스트 비저장
- [ ] `learning_signal_events`에서 다음 원문 텍스트가 저장되지 않음을 확인:
  - 사용자 raw answer text
  - question 원문 text
  - reference 원문 text
- [ ] 필요 시 샘플 레코드 다건 확인 (최소 3건 권장)

### 4-2. 1차/2차 모드 분리 검증
- [ ] 이벤트/레코드에서 first/second 모드가 구분 저장됨
- [ ] 교차 오염(1차 행동이 2차로 기록되거나 반대 케이스) 없음

### 4-3. 관리자 verify 엔드포인트 로그 안전성
- [ ] admin verify endpoint 응답이 로그-세이프(log-safe) 형식임
- [ ] 응답에 민감 원문/내부 디버그 필드가 노출되지 않음

---

## 5) UX 점검

### 5-1. CTA 레이아웃 안정성
- [ ] 주요 CTA 문구가 어색하게 줄바꿈/랩핑되지 않음
- [ ] 모바일/데스크톱 주요 해상도에서 버튼 가독성 유지

### 5-2. CTA 의미 일치
- [ ] Primary CTA 라벨과 실제 이동 목적지가 일치
- [ ] “다음 행동”이 모호하지 않음

### 5-3. 내부 용어 비노출
- [ ] 학습자 화면에서 다음 내부 용어가 보이지 않음:
  - review queue
  - Gemini
  - API
  - raw field names (DB/개발 필드명)
