# S233A AI Answer Review runtime

This lane implements the personal learner, AI-only Answer Review cascade against the frozen S233 contract. It performs deterministic checks first, calls exactly one subject grader, calls the critic only for a critical finding, uncertainty, or actual deterministic/grader disagreement, and abstains when the result remains unresolved. Human approval is not a request-path stage.

The runtime binds each review to the authenticated learner owner reference; review, root submission, attempt, input version, immutable Answer Pack and snapshot; every frozen evaluator and persistence schema version; the normalized-input fingerprint; and the complete reveal history. Learner answer text, OCR text, provider bodies, credentials, and direct user identifiers are excluded from review metadata, errors, telemetry, idempotency rows, evidence records, Queue, and Today linkage.

Persistence uses learner-scoped RLS plus idempotent claim and compare-and-swap transition RPCs. Authenticated clients have learner-owned read access but no direct table mutation or RPC execution; the server re-verifies the cookie owner before its existing service-role client invokes the closed RPC with that exact owner. The claim RPC also reserves one active initial or retry request, so concurrent duplicates return a retryable in-progress result before any additional provider call. Every revision is appended to `s233a_answer_review_revisions`. A terminal transition is immutable and atomically writes Lane A evidence, the existing `personal_concept_nodes` event lineage, and exact canonical `review_queue_items` and `action_seeds` linkage. Restrictive write policies reserve only the `s233a_answer_review` Queue/Today namespace for that RPC while preserving learner reads and non-S233A workflows. Rewrite/regrade attempts retain the root submission and predecessor review lineage; a verified correction appends `corrected` evidence without replacing the original `detected` record. Lane A emits only `detected`, `corrected`, or `uncertain`.

The Answer Pack loader accepts only a `verified_learning_reference` with matching S214/S215 release records and no unresolved blocker. It fails closed if the requested immutable pack, registry context, release proof, source snapshot, or evaluation reference cannot be validated. Because the frozen pack contract has no skill-to-claim mapping, a pack with multiple source anchors deterministically abstains before learner text reaches a provider rather than assigning anchors by position. The reused structure grader likewise binds learner evidence only when the input has one unambiguous segment and otherwise abstains instead of inventing a locator. The API reuses the existing Answer Review entitlement check before provider work and records the existing usage event only when a non-replayed request actually called the primary grader; the event metadata contains no learner or review identifiers. OCR and model behavior are faked in tests; no live provider or browser is part of focused validation.

## Migration operations

`20260721060237_s233a_answer_review_persistence.sql` is forward-only and additive. Before applying, capture a database backup and verify that the prerequisite Review OS tables and `transition_personal_concept_node_v1` exist. Apply in a staging project first, then verify RLS with two authenticated test users, duplicate request replay, stale CAS rejection, and terminal replay.

Rollback is allowed only before learner traffic or after exporting and reconciling every S233A review, revision, evidence record, Queue item, Today seed, and concept transition. An operator must first disable the S233A route, prove that no dependent records remain, and use a separately reviewed forward migration to retire the two RPCs and three S233A tables. Never remove the migration from history or delete learner evidence in place.
